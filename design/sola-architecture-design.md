**Sola**，这个名字选得极好。既暗合了单源渲染（Single Source）的技术架构，又透着一种剥离繁杂、回归纯粹书写本质的静谧感。

# Sola 架构与开发设计方案文档 (V1.0)

## 一、 产品定位与核心设计哲学

**Sola**（源自拉丁语，意为“独白/唯一”）旨在解决 Electron 编辑器臃肿、响应延迟的痛点，通过极致的原生技术栈重塑 Markdown 书写体验。

- **唯一源 (The Sola Source)**：摒弃传统的双栏预览模式，采用块级无缝切换（WYSIWYG）机制。文档本身即是代码，也是最终的渲染呈现。
- **极致性能**：基于 GPU 加速，启动时间控制在 100ms 内，支持十万行级别超大文件无感滚动。针对现代桌面协议（如原生 Wayland 环境）进行深度适配，消除中间层损耗。
- **本地与纯净 (Local-First)**：数据完全留存本地，零遥测，不内置任何臃肿的浏览器内核。
- **诗意与极客的平衡**：提供极速的渲染反馈、全局结构化动态主题，并原生内置出版级的 PDF 与高质量排版长图导出能力。

## 二、 核心技术选型 (Tech Stack)

| **领域**       | **核心技术栈**        | **深度选型考量**                                             |
| -------------- | --------------------- | ------------------------------------------------------------ |
| **GUI 框架**   | **GPUI**              | GPU 级加速渲染，天生适合做高性能文本编辑器，响应速度极快。   |
| **异步调度**   | **Tokio**             | 榨干多核 CPU 算力，将解析、渲染等重计算任务丢入后台，主线程永不阻塞。 |
| **AST 解析**   | **pulldown-cmark**    | Rust 生态最快且合规的 CommonMark/GFM 解析器。                |
| **语法高亮**   | **Tree-sitter**       | 增量式语法树解析，完美对接动态主题系统，实现亚毫秒级多语言高亮。 |
| **排版与公式** | **Typst**             | 纯 Rust 高性能排版引擎。既负责数学公式转 SVG，也作为原生 PDF 的底层导出核心。 |
| **图表渲染**   | **Deno Core + resvg** | 离线无头 JS 运行时解析 Mermaid，`resvg` 转为位图或矢量路径交由 GPUI 绘制，保持原生纯净性。 |

| **主题引擎**   | **Serde + TOML**      | 采用结构化配置替代 CSS，实现零计算成本的毫秒级主题热切换。   |

## 三、 核心架构设计

### 1. 块级双态渲染引擎 (Dual-State Engine)

Sola 的核心并非一个简单的富文本框，而是基于状态机的动态文档模型。

- **源码态 (Focused)**：光标落在特定区块（Block）内部时，该区块退化为带 Tree-sitter 高亮的原始 Markdown 文本（如显示 `## Sola 架构`）。
- **渲染态 (Blurred)**：光标移出后，瞬间触发 GPUI 局部重绘，结合当前全局主题渲染为格式化排版的 UI 节点。

### 2. 受控的 HTML 适配器 (The Adapter Engine)

为了兼容其他平台产生的历史文档，Sola 采用“白名单提取 + 属性映射”机制，坚决不引入 DOM 树和层叠样式表引擎：

- **轻量提取**：利用自定义 Tokenizer 提取 `<img>` 的 `width` 属性，或 `<span>` 中的内联 `color`、`font-size` 等安全样式。
- **原生映射**：将提取到的属性转化为 GPUI 的 `TextStyle` 覆盖层（Override Stack）或底层的布局约束（Constraints）。
- **优雅降级**：遇到无法解析的复杂 HTML 标签，自动降级为“代码块”形态显示，确保排版不崩溃、软件不死锁。

### 3. 全局动态主题系统

彻底抛弃 CSS，采用 TOML 结构化配置。

- 在 GPUI 的全局上下文中维护 `CurrentTheme`（包含调色板、字号缩放比例、语法高亮映射表）。
- 组件渲染闭包实时读取内存中的 16 进制颜色指针，没有 CSS 的层叠冲突计算开销，支持随操作系统深/浅色模式无缝自适应切换。

### 4. Model-View 深度解耦与 UI 虚拟化 (GPUI Idiomatic)

为了支撑十万级文件的大型工程与流畅的 UI 响应，Sola 在架构上严格遵循 Model-View 分离原则：
- **Entity 模型层 (`Worktree` / `Workspace`)**：通过 `notify` 在独立的后台线程监听系统级文件 I/O 变更，维护完整的项目数据快照，通过事件订阅 (`cx.emit`) 驱动视图，杜绝 UI 阻塞。
- **UI 虚拟化 (`gpui::list`)**：侧边栏等长列表组件采用虚拟化渲染，无论目录树多深、文件多少，永远只绘制屏幕可见的少数 DOM 节点，保持极低的内存占用与满帧的滚动体验。

### 5. 原生菜单与对话框系统 (Native Interop)

Sola 不重复造轮子，而是深度融合操作系统级交互：
- **Global Application Menu**：通过 `cx.set_menus` 构建符合 macOS/Windows/Linux 习惯的原生顶部菜单栏，实现快捷键的全局一致性。
- **系统级对话框**：当触发 `Open` 动作时，调用 `cx.prompt_for_paths` 唤起操作系统原生的文件/目录选择器，支持自由穿梭于全局文件系统。

------

## 四、 导出流水线 (Export Pipeline)

Sola 提供独立于渲染器的离线导出流水线：

- **出版级原生 PDF**：通过 `Markdown -> Typst -> PDF` 的链路。注入当前编辑器主题的样式配置，生成公式完美、文本可选中、不失真的纯矢量 PDF。
- **排版级长图导出**：利用 `tiny-skia`，根据用户设定的宽度在内存中绘制虚拟画布。支持 2x/3x 高清缩放，并可自动包裹 macOS 风格的圆角阴影。

## 五、 开发规划与里程碑 (Roadmap)

项目采用敏捷迭代策略，按五个核心阶段推进：

### Phase 1: 基础设施与骨架搭建

- 初始化 Rust 工程，跑通跨平台编译配置。
- 构建自定义 `Editor` 组件，接管键盘输入事件，实现基础的光标逻辑与 Undo/Redo 系统。
- 集成 `pulldown-cmark`，实现后台 AST 的增量解析。

### Phase 2: 双态引擎与样式适配

- 设计 `BlockElement` Trait，完成段落、标题、列表等基础组件的 Focused/Blurred 状态翻转逻辑。
- 实现 HTML Tokenizer 适配器，处理图片缩放与内联文本颜色映射。
- 跑通基于 TOML 的全局主题热切换引擎原型。

### Phase 3: 沉浸体验与高阶组件

- 全面集成 Tree-sitter，完成主流编程语言的高亮映射。
- 实现自定义 Grid 布局以支持原生表格渲染。
- 实现本地图片文件的拖拽识别、相对路径拷贝与异步加载机制。

### Phase 4: 重载图形引擎与导出系统

- 集成 Typst 实现数学公式渲染，并在前端经由 resvg 绘制。
- 搭建轻量级 JS 运行时，实现 Mermaid 流程图的异步离线生成与骨架屏占位。
- 完成基于 Typst 的原生 PDF 导出模块，以及基于 tiny-skia 的长图生成模块。

### Phase 5: 平台特化与持续交付

- 接入本地文件系统，实现带缓存的侧边栏文件树与外部变更监听（基于 `notify`）。
- 针对 Wayland 桌面环境的窗口透明、圆角与原生阴影进行精细化适配。
- **自动化发布流部署**：在 CI/CD 流水线中，**完全摒弃硬编码版本配置文件，统一通过提取 Git Tags（如 `v1.0.0`）作为单一事实来源进行自动编译、打包构建与多平台分发。**
